WhatCable for Windows

USB-C, Thunderbolt, and video-cable diagnostics — now on Windows 10 & 11.

WhatCable for Windows is a clean-sheet rewrite of the macOS app for Windows 10 22H2 and Windows 11. It explains what each USB-C port is actually doing — link speed, charging state, and (on supported PCs) the power contract and cable e-marker — in plain English. It also adds a brand-new video-cable diagnostic for HDMI, DisplayPort, and USB-C-as-DisplayPort outputs that has no macOS equivalent.

Download installer Get the MSIX package CLI docs

Free. Windows 10 22H2 or Windows 11 (widgets need Windows 11 22H2+). x64 & ARM64. No driver install.

What you get

  • Tray app. A system-tray popover that lists every USB-C port: what's plugged in, the negotiated link speed, the charging state, and — on UCSI-capable PCs — the power contract and decoded cable e-marker.
  • Video-cable diagnostic (new). For every HDMI / DisplayPort / USB-C-DP output it shows the connector, the active mode, the display's maximum capability, and a "what's limiting the link" verdict derived from EDID / CTA-861 / DisplayID.
  • Command-line tool. whatcable-cli.exe emits JSON that's schema-compatible with the macOS CLI, plus extra video and thunderbolt sections.
  • Windows 11 widgets. Small / medium / large Adaptive Card widgets for the Widgets Board and the lock screen.
  • Pro tier. Pin diagrams, a liquid-detection indicator, and a live system-wide Power Monitor graph, unlocked with an offline licence key.

Install

Installer (recommended)

Download WhatCable-Setup-<version>.exe from the Releases page and run it. It installs the tray app and the CLI, and adds a Start-menu entry. Launch-at-login is controlled inside the app under Settings → Launch at login.

MSIX package

Prefer a packaged install? Download WhatCable-<version>.msix, double-click it, and confirm in the App Installer dialog. The MSIX is required for the Windows 11 widgets, because they're registered through the package manifest. The package is code-signed; on a machine that doesn't yet trust the certificate, export the signing certificate as a Base-64 X.509 .cer from the package's digital-signature details and install it into Local Machine → Trusted People first.

From the command line

whatcable-cli.exe prints a JSON snapshot of every USB-C port, the power adapter, the Thunderbolt chain, and connected displays. The video and thunderbolt sections are Windows-only additions; pass --legacy to drop them for tools that still consume the macOS schema. 

> whatcable-cli snapshot
# macOS-compatible schema only:
> whatcable-cli snapshot --legacy
# pipe into jq:
> whatcable-cli snapshot | jq .ports[0].dataLink

FAQ

Why no per-lane Thunderbolt speeds?
There is no public Windows API that reports per-lane Thunderbolt link speeds. We enumerate the device chain via SetupAPI and label each entry honestly with unavailable_reason: per_lane_speeds_unavailable_on_windows rather than guessing. macOS exposes this through Apple-only IOKit services that Windows has no equivalent for.
Why does PD / charger / cable info require a UCSI PC?
Power-delivery contracts, charger PDOs, and cable e-marker data come from UCSI (USB Type-C Connector System Software Interface) — a firmware interface the PC's USB-C controller must expose. Most laptops from 2019 onwards have it; many desktops and older laptops don't. Without UCSI those rows show ucsi_not_supported. The USB device list and the video diagnostic don't rely on UCSI and work everywhere.
The cable e-marker is blank even on a UCSI PC. Why?
Windows only reads the e-marker after the controller negotiates above 3 A, so a low-wattage charger may never trigger the read. Try a higher-wattage USB-C charger.
Why does the video diagnostic say "unable to determine"?
That means the display's EDID couldn't be read — some virtual displays, KVMs, and capture devices don't expose one. The connector and active mode still show; the bottleneck verdict needs EDID / CTA-861 / DisplayID data.
Where are the widgets?
Widgets require Windows 11 22H2+ and the MSIX install, which registers the widget provider through the package manifest. The unpackaged installer build ships the tray app and CLI but not the widgets.
Is it the same app as macOS?
No — it's a clean-sheet WinUI 3 / C# rewrite, because the macOS app depends on Apple-only IOKit services with no Windows equivalent. It reuses the same data models, decoders, and JSON shape, so downstream tooling keeps working across platforms.
On a Mac? See the macOS docs and the CLI docs. WhatCable for macOS is a menu-bar app for Apple Silicon.
